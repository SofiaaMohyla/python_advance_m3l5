# python_advance_m3l5

Технічне завдання: Система керування студентськими клубами
Мета проекту:
Створення системи керування студентськими клубами для управління студентами, клубами, заходами та учасниками.

У процесі виконання ви маєте створити власний репозиторій на GitHub. В якості рішення - надайте ментору посилання на ваш репозиторій.

Використовуйте коміти змін та Pull Requests для більш коректної роботи над проектом.

Домашня частина
Модель "Студент" (Student):
Забезпечте можливість додавання, редагування та видалення студентів.
Поля: ім'я, прізвище, email.
Модель "Клуб" (Club):
Забезпечте можливість додавання, редагування та видалення клубів.
Використовуйте ForeignKey для встановлення зв'язку між клубом та студентом-президентом.
Використовуйте ManyToManyField для встановлення зв'язку між клубом та учасниками.
Поля: назва, опис, президент (ForeignKey на студента), учасники (ManyToManyField на студентів).
Модель "Заходи" (Event):
Забезпечте можливість додавання, редагування та видалення заходів.
Використовуйте ForeignKey для встановлення зв'язку між заходом та клубом.
Використовуйте ManyToManyField для встановлення зв'язку між заходом та учасниками.
Поля: назва, дата, місце проведення, клуб (ForeignKey на клуб), учасники (ManyToManyField на студентів).
Частина для розробки на уроці
Модель "Учасники заходів" (EventAttendees):
Забезпечте можливість додавання, редагування та видалення учасників заходів.
Використовуйте ForeignKey для встановлення зв'язків між заходом та студентами.
Поля: захід (ForeignKey на захід), студент (ForeignKey на студента).

Користувач вводить дані про студента (ім'я, прізвище, email).
Перевірка на унікальність та збереження у базі даних.
Додавання клубу:
Користувач вводить дані про клуб (назва, опис, президент).
Перевірка наявності президента та збереження у базі даних.
Додавання заходу:
Користувач вводить дані про захід (назва, дата, місце проведення, клуб).
Перевірка наявності клубу та збереження у базі даних.
Додавання учасника до заходу:
Користувач вводить дані про учасника (захід, студент).
Перевірка наявності заходу та студента та збереження у базі даних.
